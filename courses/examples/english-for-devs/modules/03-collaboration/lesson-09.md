# Code Reviews

## Pull Requests (PR)

- **Pull Request / Merge Request**: Proposing changes to the main codebase.
- **Reviewer**: Someone who checks the code.
- **Assignee**: The person responsible for the PR.
- **Draft**: A PR that is not ready for review yet.

## Feedback Terms

- **LGTM**: "Looks Good To Me" (ready to approve).
- **Nit / Nitpick**: A minor, non-blocking suggestion.
- **Approve**: Officially accept the changes.
- **Request Changes**: Block merging until issues are fixed.
